# Changelog

프론트엔드 주요 변경 이력을 날짜 기준으로 기록합니다.

## 2026-04-06

### Changed

- 헤더 GNB 설정 모달을 실제 헤더와 더 비슷한 한 줄형 PC 미리보기 + 아래쪽 모바일 미리보기 구조로 정리하고, 상위 메뉴는 헤더 바에서, 하위 메뉴는 드롭다운 패널에서 직접 드래그해 순서를 바꾸면서 drag clone과 드롭 영역 강조까지 같은 화면에서 확인할 수 있게 했습니다.
- `/adminpage/home-builder` 상단 작업 도구 영역에서 기존 GNB 관리 모달을 바로 열 수 있게 연결해, 홈 빌더 안에서 헤더 메뉴 설정까지 함께 수정하고 즉시 반영할 수 있도록 정리했습니다.
- `/adminpage/home-builder`에서 모듈 팔레트 드롭 시점에 브라우저별 `dataTransfer` 식별값이 비어 블록이 추가되지 않던 문제를 보완해, 팔레트 타입을 상태와 `text/plain` fallback으로 함께 관리하도록 정리했습니다.
- `/adminpage/home-builder`에서 `react-beautiful-dnd`가 불안정하던 팔레트→캔버스 복사 드래그 경로를 네이티브 드래그로 분리해, 모듈 팔레트의 블록을 다시 컬럼에 바로 끌어 넣을 수 있도록 복구했습니다.
- `/adminpage/home-builder`에서 캔버스 영역이 더 크게 보이도록 빌더 전체 폭을 한 번 더 확장하고, 좌측 패널 폭과 캔버스 내부 패딩을 줄였으며, 전역 `section` 패딩이 `builderSurface`에 남아 있던 부분도 함께 정리했습니다.
- `/adminpage/home-builder`의 캔버스 블록 미리보기를 `compactPreview` 축소 렌더 대신 일반 미리보기 크기로 보여주도록 바꿔, 실제 캔버스가 더 넓고 크게 보이도록 조정했습니다.
- `/adminpage/home-builder`의 최대 가로폭을 확장하고 좌측 패널 폭을 줄여, 캔버스 미리보기 영역이 더 넓게 보이도록 조정했습니다.
- `/adminpage`의 전역 섹션 폭 해제 규칙이 히어로 카드(`관리 대시보드`) 내부 패딩까지 덮어쓰던 문제를 바로잡아, 카드 내부 여백은 유지하면서 관리자 전용 폭 설정만 적용되도록 조정했습니다.
- `/adminpage`가 전역 `main { padding-top: 8rem; }` 규칙까지 중복 적용받던 문제를 리셋해, 관리자 대시보드의 상단 패딩과 헤더 간격이 어색하게 벌어지지 않도록 조정했습니다.
- `/adminpage` 내부 섹션이 전역 `section { max-width: 110rem; }` 규칙에 눌려 실제 폭이 넓어지지 않던 문제를 해제해, 관리자 전용 레이아웃 폭 설정이 실제로 적용되도록 수정했습니다.
- `/adminpage`의 전체 가로폭과 `질문 및 구독 관리` 섹션 카드 비율을 조정해 구독상품 관리 표 헤더가 좁은 폭 때문에 줄바꿈되지 않도록 보완했습니다.
- `/adminpage` 사용자, 구독, 스케줄, 질문 관리 표의 헤더를 한 줄로 유지하고 정렬 화살표와 라벨 정렬이 어긋나지 않도록 공통 테이블 헤더 스타일을 정리했습니다.
- `/adminpage`에서 GNB 설정을 저장하면 커스텀 갱신 이벤트를 통해 새로고침 없이 PC 헤더와 모바일 메뉴에 즉시 반영되도록 수정했습니다.
- `/adminpage` GNB 관리 모달에서 상위/하위 메뉴의 위로/아래로 이동 시 `order` 값도 함께 다시 계산해, 저장 후에도 실제 메뉴 위치가 바뀐 순서대로 유지되도록 수정했습니다.

## 2026-04-05

### Changed

- Google 로그인을 implicit token 방식에서 authorization code 콜백 방식으로 전환하고, `/googlelogin` 실패 시 무한 로딩 대신 명확한 오류와 로그인 복귀 흐름을 표시하도록 정리했습니다.
- 소셜 온보딩 추가정보 입력에서 앱 access token을 URL query string으로 넘기지 않고, 세션 기반 임시 상태를 통해 `/signup`에서 이어받도록 정리했습니다.
- 소셜 로그인 API 실패가 중간에서 삼켜지지 않도록 에러 처리를 정리해 콜백 페이지에서 실제 실패 원인을 사용자에게 보여주도록 보완했습니다.
- `/adminpage/home-builder`의 캔버스 미리보기가 전역 `section` 최대폭 제한에 다시 묶이지 않도록 정리해, 좌측 패널 옆 남은 영역을 전부 채우게 했습니다.
- `/adminpage/home-builder`의 행/블록 드래그를 브라우저 기본 HTML5 drag/drop 대신 `react-beautiful-dnd` 기반 정렬로 바꿔, 팔레트에서 캔버스로 끌어 넣거나 캔버스 안에서 재배치할 때 더 부드럽게 움직이도록 정리했습니다.
- `/adminpage/home-builder`의 기존 홈 섹션 블록 프리뷰를 placeholder가 아니라 실제 홈 섹션 컴포넌트 렌더로 바꿔, 기능은 막으면서 겉모양은 실제 홈과 최대한 동일하게 보이도록 정리했습니다.
- `/adminpage/home-builder`에서 모듈, 행, 블록을 드래그할 때 마우스 아래에 실제 드래그 clone이 따라다니도록 `renderClone` 기반 미리보기를 추가했습니다.
- `/adminpage/home-builder`가 첫 블록 선택 때문에 자동으로 `속성` 탭으로 넘어가던 동작을 제거해, 진입 시 `모듈 팔레트`가 기본으로 바로 보이게 정리했습니다.

## 2026-04-04

### Added

- `/adminpage`에서 데스크톱 헤더와 모바일 메뉴를 함께 수정할 수 있는 GNB 관리 카드와 메뉴 편집 모달을 추가했습니다.
- `/question` 목록과 `/question/[id]` 상세에서 어드민 전용 작성자 표시를 추가했습니다.
- `/course/[id]`, `/course/player/[id]`에 수강 진도율, 이어보기, 처음부터 보기 UI를 추가했습니다.
- `/course/write`에서 유튜브 링크 외에 동영상 파일 업로드 레슨 작성/수정을 지원하도록 확장했습니다.
- `/tutoring` 관리자 메일 양식에서 고정 수신 이메일과 발송 대상을 설정할 수 있게 했습니다.
- `/adminpage` 사용자 관리와 스케줄링 관리에서 제목과 본문을 직접 입력하는 어드민 메일 보내기 모달을 복구했습니다.
- `/adminpage/home-builder`에 홈 레이아웃 편집기와 드래그 앤 드롭 기반 미리보기 편집 흐름을 추가했습니다.
- 구독 플랜의 질문, 튜터링, 강의 권한을 `활성 구독 + 무한 토글` 기준으로 해석하는 프론트 유틸과 상태를 추가했습니다.
- 로그인 응답에서 refresh token을 함께 저장하고, 다른 기기 재로그인으로 세션이 만료되면 프론트에서 자동 로그아웃시키는 처리를 추가했습니다.
- `/adminpage`에 합격 예측 계산기 추천학교 설정 카드와 점수대/국가별 학교를 수정하는 관리자 모달을 추가했습니다.
- `/adminpage` 합격 예측 계산기 모달에서 그룹명, 그룹별 선택 가능 수, 과목, SL/HL 허용 규칙까지 수정할 수 있게 확장했습니다.
- `/adminpage` 사용자 상세 모달에 학생별 강의 진도, 질문/튜터링 사용량, 활성 구독 잔여량, 마지막 접속 시간을 보는 사용량 섹션을 추가했습니다.

### Changed

- 헤더 GNB와 모바일 사이드 메뉴가 서버 설정을 우선 읽고, 설정이 없을 때만 기존 하드코드 메뉴를 fallback으로 사용하도록 정리했습니다.
- `/aitools/coaching`의 영어 과목명을 `Language A English`로 바로잡고, 영어 모드 안내 문구 오타도 함께 수정했습니다.
- `/question`, `/mypage`, `/adminpage` 질문 목록 페이지네이션을 서버 `pageInfo` 기준으로 정리했습니다.
- `/course/player/[id]` 좌측 커리큘럼 토글과 플레이어 UI 가독성을 보완했습니다.
- `/course/player/[id]`에서 YouTube 레슨 전환 시 플레이어 인스턴스를 재사용하도록 정리하고, 강의 목차 드롭다운 레슨 가시성을 함께 보완했습니다.
- 홈 빌더의 블록 삽입/재정렬 UX를 placeholder와 밀림 동작이 보이도록 반복 개선했습니다.
- 여러 보드, 관리자, 강의, 자료실 화면의 디자인과 레이아웃을 전반적으로 정리했습니다.
- `/adminpage` 구독상품 관리와 사용자 구독 관리 화면을 `premium` 텍스트 대신 질문/튜터링/강의 수량과 무한 토글 중심으로 재구성했습니다.
- `/question`, `/tutoring`, `/course/[id]`, `/course/player/[id]`, `/mypage`, `/subscribe`가 `activeSubscribeInfo` 기준으로 실제 권한과 잔여 횟수를 반영하도록 정리했습니다.
- 로그인/토큰 재발급 흐름을 refresh token 기반으로 바꿔 한 계정의 새 로그인 이후 예전 세션이 더 이상 재발급되지 않도록 정리했습니다.
- 세션 만료 alert가 여러 번 쌓이던 문제를 막고, 한 번의 확인만으로 로그인 화면으로 이동하도록 정리했습니다.
- `/board/calculator`가 백엔드 추천학교 설정을 우선 읽고, 서버 설정이 없을 때만 기존 하드코드 데이터를 fallback으로 사용하도록 정리했습니다.
- `/board/calculator`의 그룹/과목 하드코드를 제거하고, 서버 계산기 설정의 그룹 규칙을 우선 사용하도록 정리했습니다.
